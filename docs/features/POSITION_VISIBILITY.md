# ポジション表示/非表示機能 仕様書

## 1. 概要

同一ピッチ（異弦同音）のポジションを個別に表示/非表示できる機能。
例: Gメジャースケールの「1弦3フレット」と「6弦3フレット」は同じ音名だが、
任意のポジションだけを表示/非表示にできる。

---

## 2. 用語

- **異弦同音**: 異なる弦・フレットだが同じ音名（オクターブ違いを含む）のポジション
- **表示ポジション集合 (`visiblePositions`)**: 表示する位置を明示列挙したもの。`null` は「全表示（既定）」
- **非表示ポジション**: アクティブだが表示集合に含まれない位置（薄く表示・印刷では除外）

---

## 3. 機能要件

### 3.1 操作方法（スケールエディター）

- 指板上のドットを**長押し（または専用モードで通常タップ）** → 表示/非表示をトグル
- 非表示にしたポジションは**薄くフェード表示**（完全に消えない）
  - 薄く見せることで「非表示にしてある」ことをユーザーが認識できる
  - opacity: 0.25 程度

### 3.2 表示モード切り替え

```
指板上部にトグルボタン:
[ ポジション編集モード ON/OFF ]
```

- ON 時: ドットタップで表示/非表示トグル
- OFF 時: 通常操作（度数の選択等）

### 3.3 一括リセット

- 非表示ポジションを全て表示に戻す「リセット」ボタン

### 3.4 印刷時の扱い

- 非表示ポジションは印刷でも**表示されない**（viewBoxから除外）
- ソングブック、ソングファイルがめんでも非表示ポジションは表示されない

---

## 4. データ構造

スケールの状態に `visiblePositions` フィールドを追加。
**「表示するポジション」を明示的に持つ**（非表示の差分ではない）。

```js
// 既存の scale オブジェクト
{
  rootIndex: 0,
  activeDegrees: new Set([0,2,4,7,9]),
  // ...existing fields...

  // 新規追加: 表示するポジション
  //   null = 未カスタマイズ → アクティブな全ポジションを表示（既定）
  //   Set  = この集合に含まれる位置だけ表示（他は非表示=薄く表示）
  visiblePositions: new Set([
    'g3s0',  // string 0(0始まり), fret 3 → "g{fret}s{string}"
    'g5s1',
  ])
}
```

キー形式: `g{fret}s{string}` （fret=フレット番号, string=弦番号 0始まり）。
1キー = 指板上の1ドット（弦×フレット）を一意に指す。

### なぜ「表示する位置」を持つか（非表示の差分ではない理由）

**カスタムスケールがある**ため、必ずしも標準スケール（メジャーペンタ／単一度数 等）とは
限らない。「大半は表示」という前提が成り立たず、特定の運指の型だけ出す使い方では
**表示する数の方が少ない**。よって表示集合を直接持つほうが自然でサイズも小さい。

### 描画ロジック

`visiblePositions` は**常に「表示中の位置の実体集合」**として同期管理する。

- **表示 = (現在アクティブな位置) ∩ visiblePositions**。集合に無いアクティブ位置は
  **非表示（薄く表示・印刷では除外）**。
- `visiblePositions === null` は「未設定（旧データ等）」のフォールバックで、全アクティブ表示扱い。
  通常の編集操作では必ず実体集合に**材化（マテリアライズ）**する。

### 集合の更新ルール（確定）

| 操作 | visiblePositions の更新 |
|------|------------------------|
| **プリセット（スケール/コードトーン）を選択** | **一旦クリアし、選んだプリセットのアクティブ全ポジションで再構築**（＝そのとき表示される全位置を再登録） |
| **カスタム編集で度数を有効化** | その度数の全ポジションを**追加**（表示） |
| **カスタム編集で度数を無効化** | その度数の全ポジションを**削除**（非表示） |
| **個別ポジションをタップ（異弦同音）** | そのキー1つを**トグル**（表示↔非表示） |
| **「リセット」** | 現在アクティブな全ポジションで再構築（プリセット選択と同じ） |

→ つまり「度数構成が変わったら表示集合を作り直し、その後の個別タップで間引く」。
プリセット切替・度数トグルで**前のカスタム間引きは破棄**される（構成が変わったので作り直すのが直感的）。

### 実装メモ（保存経路で必ず対応）

- `state/snapshot.js` の `cloneEditAsSnapshot()` に
  `visiblePositions: edit.visiblePositions ? new Set(edit.visiblePositions) : null` を追加する
  （現状は未対応。色 `degreeColors` と同様にスケール単位で複製する）。
- localStorage / D1(ソングブック) は JSON なので **Set ⇄ Array** を変換する:
  保存時 `visiblePositions ? Array.from(set) : null`、読込時 `Array.isArray(v) ? new Set(v) : null`。
- `persist.js` の読込サニタイズで `/^g\d+s\d+$/` に合致しないキーは破棄（破損データ対策）。
  配列でも null でもない不正値は `null`（全表示）にフォールバック。
- 楽器（guitar/bass）で弦数が違うため、範囲外の弦・フレットのキーは描画時に無視する。

## 5. 受け入れ条件

| # | 条件 |
|---|------|
| AC-01 | ポジション編集モードON時、ドットタップで非表示にできる |
| AC-02 | 非表示ポジションは薄く（opacity低下）表示される |
| AC-03 | もう一度タップすると再表示される |
| AC-04 | 「リセット」で現在アクティブな全ポジションに再構築される（全表示に戻る） |
| AC-05 | 表示ポジション設定（`visiblePositions`）がlocalStorageに保存される |
| AC-06 | 印刷時に非表示ポジション（表示集合外）は出力されない |
| AC-07 | ソングブック保存/読み込み・共有時に表示ポジション設定も保持される |
| AC-08 | プリセット（スケール/コードトーン）選択で表示集合が新プリセットの全ポジションに再構築される |
| AC-09 | カスタム編集で度数を有効化→その度数の位置が表示、無効化→非表示になる |
