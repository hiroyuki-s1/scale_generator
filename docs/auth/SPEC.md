# 認証・クラウド保存 仕様書

## 1. 概要

登録スケールをクラウドに保存し、どのデバイスからでも使えるようにする。
ログイン機能は Clerk（Google OAuth）、データ保存は Cloudflare D1 を使用する。

---

## 2. ユーザーストーリー

| # | As a... | I want to... | So that... |
|---|---------|-------------|------------|
| US-01 | ユーザー | Google アカウントでログインしたい | 複数デバイスでスケールを共有できる |
| US-02 | ユーザー | 登録スケールをクラウドに保存したい | スマホ・PC どちらからでも使える |
| US-03 | ユーザー | ログアウトできる | 共有端末で安全に使える |
| US-04 | 未ログインユーザー | ログインせずに使い続けられる | アカウント登録を強制されない |
| US-05 | ログインユーザー | 印刷レイアウト設定が保存されたい | 毎回設定し直さなくて済む |
| US-06 | メール登録ユーザー | パスワードを忘れても再設定したい | ログインできなくならない |
| US-07 | ユーザー | アカウントを削除（退会）したい | 自分のデータを完全に消せる |

## 3. 機能要件

### 3.1 認証

- **ログイン方法**: Google OAuth ＋ メール/パスワード（いずれも Clerk 経由）
- **Clerk プリビルトUI を利用**: ログイン・新規登録・パスワード再設定・メール確認・
  アカウント管理は Clerk の `<SignIn>` / `<SignUp>` / `<UserProfile>` を使う（**フォームは自作しない**）
- **任意ログイン**: 未ログインでも全機能を使用できる（localStorage で動作）
- **セッション**: Clerk が管理。リロード後も維持
- **ログアウト**: ヘッダーのユーザーメニューから実行

### 3.2 データ同期

- **ログイン時**: D1 のスケールを取得して表示
- **スケール登録**: D1 に保存（ログイン時）/ localStorage に保存（未ログイン時）
- **スケール削除**: D1 から削除（ログイン時）
- **並び替え**: sort_order を更新
- **設定保存**: レイアウト設定（cols/rows/orientation）を user_settings に保存

### 3.3 未ログイン → ログイン時のデータ移行

- localStorage にスケールがある状態でログインした場合、D1 にインポートするか確認する
- ユーザーが「移行する」を選んだ場合、localStorage のスケールを D1 に追加する

### 3.4 アカウント管理（パスワード再設定・退会）

すべて Clerk 標準機能を利用し、独自実装は最小化する。

#### パスワード再設定（メール/パスワードログイン時）

- Clerk の `<SignIn>` に含まれる「**パスワードをお忘れですか？**」フローを使う
  （登録メールにリセット用コード/リンクを送る → 新パスワード設定）。**アプリ側で作らない**。
- Google ログインのみのユーザーにはパスワードが無いので対象外（再設定は Google 側）。

#### アカウント削除（退会）

- Clerk Dashboard で **self-service のアカウント削除を有効化**
  （User & Authentication → アカウント削除を許可）。
- ユーザーはヘッダーのユーザーメニュー → Clerk の `<UserProfile>` →「アカウントを削除」から実行。
  **削除確認ダイアログは Clerk が表示**する。
- 削除時に Clerk が `user.deleted` Webhook を送出 → `POST /api/webhooks/clerk` が
  その `user_id` の D1 データを**全テーブルから物理削除**（`songbooks` / `user_settings` / `shares`）。
  → [API.md](./API.md) の「Clerk Webhook」参照。
- 退会後、その端末の **localStorage のローカルデータは残る**（サーバ管理外）。
  サインアウト時にクライアントでもクリアするかは UX 判断（既定は残す＝未ログイン利用に戻るだけ）。

---

## 4. 非機能要件

- **オフライン**: ログイン済みでもオフライン時は localStorage にフォールバック（v2 対応）
- **レスポンス**: API レスポンス 500ms 以内
- **セキュリティ**: 他ユーザーのデータは参照・更新不可（Workers 側で user_id 検証）

## 5. UI 要件

### 5.1 未ログイン状態

```
ヘッダー右端:
[ ログイン ] ボタン
```

### 5.2 ログイン状態

```
ヘッダー右端:
[ ユーザーアイコン(Google) ▼ ]
    → ログアウト
```

### 5.3 ログインモーダル（Clerk `<SignIn>` を埋め込み）

```
┌─────────────────────────────┐
│  ログイン                    │
│                             │
│  [ G  Google でログイン ]    │
│  ───────── または ───────── │
│  メール    [____________]   │
│  パスワード [____________]   │
│  [ ログイン ]                │
│  パスワードをお忘れですか？   │  ← Clerk の再設定フロー
│  アカウント作成              │  ← Clerk の <SignUp>
│                             │
│  ※ ログインは任意です        │
│  　 未ログインでも全機能使用可 │
└─────────────────────────────┘
```

※ 上記は Clerk `<SignIn>` が提供する要素。レイアウトは Clerk の外観設定に従う。
　アカウント管理（パスワード変更・退会）はユーザーメニュー → `<UserProfile>`。

### 5.4 法務リンク（必須）

- ログイン/登録画面に「登録により[利用規約](../legal/TERMS.md)と
  [プライバシーポリシー](../legal/PRIVACY.md)に同意したものとみなします」を明示。
- フッター等にも両ページへのリンクを常設。
- Clerk のダッシュボード設定にも利用規約・プライバシーポリシーの URL を登録する。
- 公開ページとして配信する（例: `/privacy`・`/terms`、または静的 HTML）。

## 6. 受け入れ条件

| # | 条件 |
|---|------|
| AC-01 | 「Google でログイン」ボタンをタップすると Google の認証画面が開く |
| AC-02 | ログイン後、ヘッダーに Google アカウントのアイコンが表示される |
| AC-03 | ログイン後、登録スケールが D1 から読み込まれる |
| AC-04 | スケール登録・削除・並び替えが D1 に反映される |
| AC-05 | ログアウト後、ヘッダーが「ログイン」ボタンに戻る |
| AC-06 | 未ログインでも従来通り localStorage で全機能が使える |
| AC-07 | 異なるデバイスでログインすると同じスケールが表示される |
| AC-08 | メール/パスワードで登録・ログインできる |
| AC-09 | 「パスワードをお忘れですか？」からメール経由でパスワードを再設定できる |
| AC-10 | ユーザーメニューからアカウントを削除でき、確認後に退会できる |
| AC-11 | 退会時に D1 のソングブック・設定・共有が全て物理削除される |
