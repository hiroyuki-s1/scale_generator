# 画像（PNG）出力機能 仕様書

## 1. 概要

ソングファイル画面で、スケールの指板を **PNG 画像**として書き出す機能。

- **個別出力**: 各スケールカードの「画像」ボタン → そのスケールだけを PNG 出力
- **一括出力**: ソングファイルタブ上部の「画像一括出力」 → 登録中の全スケールを出力

印刷（PDF）とは別に、SNS 共有・資料貼り付け・スマホ保存に使える。

---

## 2. UI

### 2.1 個別出力（各スケールカード）

```
┌─────────────────────────────────────┐
│ C メジャーペンタ      [画像] [印刷] 🗑 │
│ [指板 SVG]                           │
└─────────────────────────────────────┘
```
- 「画像」ボタン → そのカードの指板を PNG でダウンロード。
- ファイル名 = スケール名（サニタイズ）＋ `.png`（例: `C メジャーペンタ.png`）。

### 2.2 一括出力（ソングファイルタブ上部）

```
┌─────────────────────────────────────┐
│ ソングファイル      [画像一括出力] ... │
├─────────────────────────────────────┤
│ ...スケールカード一覧...             │
└─────────────────────────────────────┘
```
- 登録中の全スケールを画像化。**出力形式は §3.3 参照**。
- 0件のときはボタン無効。

## 3. 技術方式（ランタイム依存ゼロ）

### 3.1 SVG → PNG 変換

指板は SVG（[src/ui/fretboardSvg.js](../../src/ui/fretboardSvg.js)）なので、ブラウザ標準APIだけで変換できる:

```
1. 対象 <svg> を XMLSerializer で文字列化
2. data URL (image/svg+xml) を <img> に読み込む
3. <canvas>（高解像度: 論理サイズ × スケール係数）に drawImage
4. canvas を白背景で塗ってから SVG を描画（PNG 透過を避け視認性確保）
5. canvas.toBlob('image/png') → URL.createObjectURL → <a download> でダウンロード
```

- **依存なし**（`XMLSerializer` / `Image` / `Canvas` / `Blob` / `URL` のみ）。
- **解像度**: 開発者ノブの**スケール係数**（例 `IMAGE_EXPORT_SCALE = 3`）で論理サイズの3倍など高精細に。
  値は [src/config.js](../../src/config.js) に定数として置く（印刷フォントと同様にノブ化）。

### 3.2 タイトルの焼き込み

画像にスケール名を含めるため、印刷と同じ **`bakePrintTitle` 方式**で SVG の viewBox 上端に
タイトル帯を足してから書き出す（[src/ui/fretboardSvg.js](../../src/ui/fretboardSvg.js)）。
書き出し後は元に戻す。→ 名前＋指板が必ず1枚の画像になる。

### 3.3 一括出力の形式

| 方式 | 内容 | 採否 |
|------|------|------|
| **個別PNGを順次ダウンロード** | 各スケール=1ファイル。ファイル名はスケール名 | ✅ **採用（確定）**。依存ゼロ・実装簡単。ブラウザの「複数ファイルのダウンロード」許可が1回出る |
| 1枚にまとめたシートPNG | 印刷グリッドのように全スケールを1枚に | ✖ 不採用（枚数多いと canvas 上限で要ページ分割） |
| ZIP にまとめる | 1ファイルで配布 | ✖ 不採用（無圧縮ZIPの自作が必要） |

→ **一括出力は「個別PNGを順次ダウンロード」で確定**。各スケールを1枚ずつ、スケール名の
ファイル名でダウンロードする。1ファイル化が必要になれば将来 ②/③ を追加検討。

## 4. 例外・エッジケース

| ケース | 挙動 |
|--------|------|
| フォントが canvas で描画されない | SVG 内の文字は**システム/インラインフォント**で描く。外部Webフォント依存を避ける（必要なら data URI で埋め込み）。要実機確認 |
| 楽器未選択（指板非表示） | 画像ボタンは無効（出力対象なし） |
| canvas が tainted（外部画像参照） | 指板SVGは図形+文字のみで外部参照なし → taint しない。将来画像を足す場合は注意 |
| `toBlob` 失敗 | トースト「画像の出力に失敗しました」 |
| 一括で大量（〜50件） | 順次ダウンロード。進捗を簡易表示。失敗分はスキップして継続 |
| ファイル名の不正文字 | スケール名を `/\:*?"<>|` 等を除去してサニタイズ。空なら `scale.png` |

## 5. 受け入れ条件

| # | 条件 |
|---|------|
| AC-01 | 各スケールカードに「画像」ボタンがある |
| AC-02 | 「画像」でそのスケールの指板が PNG でダウンロードされる |
| AC-03 | 画像にスケール名が含まれる（タイトル焼き込み） |
| AC-04 | 画像は高解像度（スケール係数適用）で出力される |
| AC-05 | 背景は白（透過しない） |
| AC-06 | ソングファイル上部に「画像一括出力」があり、全スケールを出力できる |
| AC-07 | 0件のとき一括出力ボタンは無効 |
| AC-08 | 出力失敗時にトーストでエラー表示し、他の出力は継続する |
