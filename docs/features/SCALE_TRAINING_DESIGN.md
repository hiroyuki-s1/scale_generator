# スケールトレーニング — デザイン改良ブリーフ

> 対象機能: `…` メニュー →「スケールトレーニング」（テンポ同期のスケール進行ゲーム）
> 実装: `src/ui/scaleTrainGame.js` / `src/domain/scaleGame.js`(pure・採点) / `src/styles/main.css`(`.stg-*`) / `index.html`(`#stgOverlay`)
> 方針: 本体はライト（クリーム＋オレンジ `--accent:#c0511f` / 合致=緑 `--green:#16a34a`）。チューナー等と同じ家系の見た目を保つ。
> このドキュメントは「デザインを改良してもらう」ためのブリーフ。実装者は各案を P0→P2 の順に着手。

---

## 0. ゴール（このデザインで達成したいこと）
1. **一目で分かる**：速いテンポでも「今どのスケールか／次は何か／合ってるか」が瞬時に読める。
2. **間に合う**：スケール切替の前に準備できる（先出し＋切替予告）。
3. **気持ちいい**：合致/コンボ/高ランクが報酬的に感じる（ゲームのリザルト風）。
4. **本体と地続き**：色温度・フォント・角丸・ピル形状を本体ライトに揃える。

非ゴール: 楽譜表示・MIDI入力・多重音採点（将来検討）。

## 1. 現状の画面と要素（実装済み）

### 1.1 設定画面 `#stgSetup`
- テンポ ステッパー `#stgTempo`（±5・既定120）
- ループ回数 ステッパー `#stgLoops`（±1・既定2）
- 「1スケール … 4拍で切替」情報行
- 進行チップ `#stgProgression`（`.stg-prog-chip` = `1 C メジャー` …）
- `START` `#stgStartBtn`、補足 `#stgSetupNote`（`Nスケール × Mループ = K拍`）

### 1.2 カウントイン → プレイ `#stgPlay`
- 進捗バー `#stgProgressFill`
- ループ表示 `#stgNow`（`1 / 2 ループ` / カウントイン中は「スタートまで N」）
- 現在スケール名 `#stgScaleName`（大）
- 拍ドット `#stgBeats`（4個・現在拍 on・拍頭=downbeat 緑）
- 許可音チップ `#stgAllowed`（`.stg-allowed-chip` 丸・ルート橙・弾いた音 lit 緑）
- **次スケール先出し** `#stgNext`（`つぎ` ピル＋名前＋小チップ・ルート橙）
- 判定 `#stgVerdict`（○/✕ 大）、`#stgPlayedNote`（弾いた音）
- スコア `#stgScoreOk/Ng/Combo`（正解/ミス/コンボ）

### 1.3 リザルト `#stgResult`
- `RESULT` ヘッダ、ランク `#stgRank`（S/A/B/C/D・ポップ＋色）、正答率 `#stgAccuracy`
- スタッツ表 `#stgResultStats`（正解/ミス/最大コンボ/許容）
- `もう一度` `#stgRetryBtn` / `閉じる` `#stgCloseBtn`

採点(pure): 現在スケール内=correct / 外=miss / 切替直後1拍だけ前スケール=tolerated。ランク = `rankFor(accuracy)`。

## 2. 課題（現状の弱点）

| # | 画面 | 課題 |
|---|---|---|
| C1 | プレイ | 情報がテキストで横並び。速い時に視線が散る。**「今／次／判定／スコア」の階層が弱い**。 |
| C2 | プレイ | 拍が4ドットのみ。**切替まで残り何拍か**が直感的でない（カウントダウン無し）。 |
| C3 | プレイ | 次スケールは出るが**切替が迫っても強調されない**（最後の1拍で予告が欲しい）。 |
| C4 | プレイ | 判定が ○/✕ の素朴な点滅。**ミスの痛み／コンボの快感**が薄い（画面フィードバック弱）。 |
| C5 | プレイ | 許可音が丸チップ。**本体の指板/ピアノ表現と不一致**（学習導線が切れている）。 |
| C6 | リザルト | ランク＋数値のみ。**達成感・内訳（スケール別）・もう一回したくなる導線**が弱い。 |
| C7 | 設定 | 進行チップが素。**所要時間・難易度・並び替え**などの情報/操作が無い。 |
| C8 | 全体 | モーション言語が断片的。**ゲームとしてのトーン**（色・動き・コピー）が未統一。 |

## 3. 改良案（優先度つき）

### P0（最優先・体験の芯）

**A. 切替カウントダウン＆次スケール予告の強調**（C2/C3）
- 拍ドットの近くに「**切替まで N拍**」を表示。残り1拍で `#stgNext` を**拡大＋パルス**、拍頭の緑フラッシュと同期。
- 案: 現在スケール名の脇に小さなリング/バーで「このスケールの4拍の進み」を可視化（4分割の円 or 横バー）。
```
        C メジャー            ← 現在(大)
   ●  ●  ●  ◐   切替まで1拍   ← 拍 + 残り
   [C][D][E][F][G][A][B]      ← 許可音
   ┌───────────────────────┐
   │ つぎ ▶ A マイナー  ⟳   │  ← 残り1拍で枠が光る/拡大
   │ [C][D][E][F][G]&#91;A&#93;[B] │
   └───────────────────────┘
```

**B. 判定フィードバックの強化**（C4）
- correct: チップ点灯＋**コンボ数のポップ**（コンボが伸びるほど色・サイズ強調、例 10/20/30 で段階演出）。
- miss: **画面端の赤フラッシュ＋軽い横シェイク**、コンボ0へのリセットを視覚化。
- `prefers-reduced-motion` で簡易化（点灯/色のみ）。

**C. リザルトの達成感**（C6）
- ランクを**大きく中央＋紙吹雪/きらめき**（S/A は派手、C/D は控えめ）。
- **正答率リング**（円形プログレス）＋ best更新バッジ（localStorage に自己ベスト保存→「自己ベスト更新！」）。
- **スケール別の内訳**（各スケールの正解/ミスを小リストやヒートで）。
- 「もう一度」を主、「設定を変えて再挑戦」を副に。

### P1（学習価値・分かりやすさ）

**D. 許可音を本体の表現に寄せる**（C5）
- 丸チップ → **ミニピアノ鍵盤**（スケール練習 `scaleTrainer` の鍵盤UIと共通化）か、**指板帯**。弾いた音をその上で光らせる。
- 度数ラベル（R/M3…）を併記（`scalePractice.classifyAgainstScale` を再利用）。学習導線が一気に強くなる。

**E. プレイ画面の情報レイアウト再設計**（C1）
- ゾーン分割: ①上=進行/ループ/残り ②中=現在スケール+許可音(主役) ③その下=次スケール ④下=判定/スコア。
- 判定は中央に大きく一瞬、スコアは常時小さく。視線の主役は「許可音」。

**F. 設定画面の充実**（C7）
- 進行チップに**所要時間の推定**（`Nスケール×Mループ×4拍×beatMs`）と**ドラッグ並び替え/一部スキップ**（任意）。
- 「カウントイン拍数」「メトロノーム ON/OFF・音量」を設定可能に（実機で要望出やすい）。

### P2（彩り・将来）
- ランクに応じた**称号/フレーバーコピー**（「完璧！」「あと一歩」）。
- **2つ先**まで薄く先出し（先読み上級者向け・トグル）。
- 効果音（正解チャイム/ミス音）※メトロノームと干渉しない設計（検出レンジ外・短尺）。
- リザルトの**シェア画像**（本体の画像出力資産を流用）。

## 4. ビジュアル方針

- **色**: 地=本体クリーム（`--bg #f7f5f1`/`--surface #fff`）。操作/強調=オレンジ（`--accent #c0511f`）。**合致/正解=緑 `--green`**、**ミス=赤 `#d1342b`**。次スケールは彩度を落として「予告」感（現状踏襲）。
- **タイポ**: 本体フォント。スケール名・ランクは極太・大。数値は等幅(`tabular-nums`)。
- **形**: ピル/丸（本体と統一）。カードは `--surface` + `--border` + 角丸14。
- **トーン**: 「練習だけど、ちょっとゲーム」。派手すぎず、合致と高ランクだけ気持ちよく光る。

## 5. モーション方針
- 拍頭（切替）で緑フラッシュ＋次枠パルス。
- correct=チップが弾むスケールポップ、combo node が伸びる。miss=赤フラッシュ＋微シェイク。
- リザルト=ランクが `scale .3→1.15→1` でポップ（既存 `stgRankPop` を発展）。
- すべて `@media (prefers-reduced-motion: reduce)` で無効化/簡易化。

## 6. コピー（文言）方針
- 短く・前向き。プレイ中ヒントは1行（現状「鳴っている音が今のスケールに入っていれば○・外れたら✕」）。
- リザルト: ランク別に一言（S=「完璧！」/ D=「まずはゆっくりから」）。

## 9. 実装メモ（着手者向け）
- 採点ロジックは `domain/scaleGame.js`（pure・テスト済み）。**UIだけ**触れば見た目改良は完結する（採点は変えない）。
- 残り拍 = `game.beatsPerScale - schedule.beatInStep - (拍内進捗)`。`scheduleAt(t,game)` から算出可。
- 許可音の鍵盤化は `scaleTrainer.js` の `.strainer-key` ロジックを共通部品に切り出すと二重実装を避けられる。
- 自己ベスト/設定追加は localStorage（既存 `sg.v1.stg*` に倣う）。ログインユーザーは `cloudSync.patchSettings` で D1 同期も可（tunerOffsets/Theme と同じ枠組み）。
- すべて e2e（`e2e/tuner.test.cjs` の STG ケース）と単体（`__tests__/domain/scaleGame.test.js`）を壊さないこと。モーションは `prefers-reduced-motion` 対応必須。
