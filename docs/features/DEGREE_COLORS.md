# 度数カラー設定 仕様書

## 1. 概要

現在は全スケール共通の1セットのカラー設定しか持てない。
これを**スケールごとの個別設定**と**一括設定**の両方に対応する。

---

## 2. 変更点

| 項目 | 現在 | 変更後 |
|------|------|--------|
| 色設定の粒度 | 全スケール共通1セット | スケールごと個別 + 一括適用 |
| 色設定UIの場所 | 色ボタン（ヘッダー） | スケールエディター内に配置 |
| 一括反映 | 色変更が全スケールに自動伝播 | 「一括反映」ボタンで明示的に適用 |

---

## 3. 機能要件

### 3.1 スケールエディター内の色設定UI

- 色設定パネルをスケールエディターに常駐表示（現在の色ボタンモーダルを移動）
- 編集中のスケールの `degreeColors` を設定する
- 変更はリアルタイムで指板に反映される

### 3.2 一括反映ボタン

```
色設定パネル内:
[ 現在の色設定をソングファイルの全スケールに適用 ]
```

- タップで確認ダイアログ:
  「現在の色設定をソングファイルの全スケール（N件）に適用します。
   各スケールの個別設定は上書きされます。よろしいですか？」
- OK → 全スケールの `degreeColors` を現在の設定で上書き

### 3.3 個別設定

- 各スケールカード（ソングファイルタブ）で「色を編集」ボタン
- タップするとそのスケールの色設定モーダルを開く
- 変更はそのスケールのみに適用

---

## 4. データ構造への影響

現在の `degreeColors` はすでにスケールごとに `saved[]` に保持されている。
`propagateColors()` による自動伝播を廃止し、明示的な「一括反映」に変更する。

```
現在: 色変更 → propagateColors() → 全スケールに自動反映
変更後: 色変更 → 編集中スケールのみ変更
        「一括反映」ボタン → 全スケールに適用
```

---

## 5. 受け入れ条件

| # | 条件 |
|---|------|
| AC-01 | 色設定UIがスケールエディター内に表示される |
| AC-02 | 色変更が編集中のスケールにのみ反映される |
| AC-03 | 「一括反映」ボタンで確認後、全スケールに色が適用される |
| AC-04 | 各スケールカードから個別に色を編集できる |
| AC-05 | 個別設定した色が印刷・ソングブック保存時も保持される |
