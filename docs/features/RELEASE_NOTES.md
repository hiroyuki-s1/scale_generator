# リリースノート表示機能 仕様書

## 1. 概要

右上の「…」メニューに「リリースノート」項目を追加し、タップすると
バージョンごとの更新内容をモーダルで表示する。
リリース内容はソースに同梱した**データファイル**で管理する。

---

## 2. 機能要件

### 2.1 メニュー項目

```
右上「…」メニュー:
  ├ フルスクリーン
  ├ 印刷
  ├ リリースノート   ← 新規追加
  └ ...
```

### 2.2 リリースノートモーダル

- タップでモーダルを開き、新しいバージョンから降順に一覧表示
- 各バージョンに「バージョン番号・日付・変更点リスト」を表示

```
┌─────────────────────────────────┐
│ リリースノート              [ × ] │
├─────────────────────────────────┤
│ v1.1.0  2026-07-01               │
│  ・ソングブック（クラウド保存）追加 │
│  ・度数カラーをスケールごとに設定可 │
│  ・異弦同音の表示/非表示機能        │
├─────────────────────────────────┤
│ v1.0.1  2026-06-05               │
│  ・印刷レイアウトの改善            │
│  ・PWA インストール対応            │
└─────────────────────────────────┘
```

---

## 3. データファイル

リリース内容は **`public/release-notes.json`** に置く（ビルドで dist に含まれる）。
ランタイムで `fetch('release-notes.json')` して読み込む（base path に追従）。

```json
{
  "releases": [
    {
      "version": "1.1.0",
      "date": "2026-07-01",
      "highlights": [
        "ソングブック（クラウド保存）を追加",
        "度数カラーをスケールごとに設定可能に",
        "異弦同音ポジションの表示/非表示機能を追加"
      ]
    },
    {
      "version": "1.0.1",
      "date": "2026-06-05",
      "highlights": [
        "印刷レイアウトの改善（モバイル/PC 個別最適化）",
        "PWA インストールに対応"
      ]
    }
  ]
}
```

- **一次ソースは JSON 1ファイル**。リリースのたびにここへ追記する
- `version` は `package.json` の version と一致させる運用（手動）
- 表示順はファイルの配列順（新しい順で記載する）

---

## 4. 未読バッジ（任意・将来）

- 最後に閲覧したバージョンを `localStorage('sg.v1.lastSeenRelease')` に保存
- 現行バージョン > 既読バージョン のとき「…」に赤バッジを出す
- リリースノートを開いたら現行バージョンを既読として保存

---

## 5. 例外処理

| ケース | 挙動 |
|--------|------|
| `release-notes.json` の fetch 失敗 | モーダル内に「リリースノートを読み込めませんでした」を表示（アプリは継続） |
| JSON パース失敗 | 同上。console.error にログ、UI はフォールバック文言 |
| `releases` が空配列 | 「リリースノートはまだありません」を表示 |
| `highlights` 欠落/非配列 | そのバージョンは見出しのみ表示（クラッシュしない） |

---

## 6. 受け入れ条件

| # | 条件 |
|---|------|
| AC-01 | 「…」メニューに「リリースノート」項目がある |
| AC-02 | タップするとバージョン一覧モーダルが開く |
| AC-03 | 新しいバージョンが上に表示される |
| AC-04 | `release-notes.json` の追記だけで内容が更新される |
| AC-05 | fetch 失敗時もアプリがクラッシュせずフォールバック表示する |
