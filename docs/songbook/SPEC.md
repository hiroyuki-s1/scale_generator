# ソングブック機能 仕様書

## 1. 概要

現在の「登録スケール」画面（複数スケールのリスト）を **ソングファイル** と呼ぶ。
ソングファイルを名前を付けてクラウドに保存する機能が **ソングブック**。

例: 「Autumn Leaves 用スケールセット」「Bossa Nova 練習セット」など、
曲・目的ごとにスケールのまとまりを保存・呼び出しできる。

---

## 2. 用語定義

| 用語 | 旧名称 | 説明 | 保存先 |
|------|--------|------|--------|
| **スケール** | （同じ） | 1つの指板設定（ルート＋度数） | localStorage |
| **ソングファイル** | 登録スケール | 複数スケールのリスト（現在の画面） | localStorage |
| **ソングブック** | （新規） | 名前付きソングファイルの集合 | Cloudflare D1 |

---

## 3. タブ構成

```
現在:
[ スケールエディター ] [ 登録スケール ]

変更後:
[ スケールエディター ] [ ソングファイル ] [ ソングブック ]
                                              ↑ ログイン時のみ表示
```

---

## 4. ユーザーストーリー

| # | As a... | I want to... | So that... |
|---|---------|-------------|------------|
| US-01 | ログインユーザー | 今のソングファイルを名前をつけて保存したい | 曲ごとのスケールセットを管理できる |
| US-02 | ログインユーザー | 保存したソングブックを一覧で見たい | 過去に作ったセットを確認できる |
| US-03 | ログインユーザー | ソングブックをタップして読み込みたい | 練習曲に合わせてすぐに切り替えられる |
| US-04 | ログインユーザー | ソングブックを削除したい | 不要になったセットを整理できる |
| US-05 | ログインユーザー | ソングブックの名前を変更したい | 後から整理できる |
| US-06 | 未ログインユーザー | ソングブックタブを見てログインを促されたい | 機能の存在を知ってログインしたくなる |

## 5. 機能要件

### 5.1 ソングファイルタブ（旧: 登録スケール）

- 機能は現在と同一（スケールの登録・削除・並び替え・印刷）
- タブ名を「ソングファイル」に変更
- タブ最上部に「現在編集中のスケール」プレビューを表示（→ 6節参照）
- ログイン時: 「ソングブックに保存」ボタンを追加（画面下部）

### 5.2 ソングブックタブ

- **未ログイン時**: ロック画面を表示（「ログインすると使えます」）
- **ログイン時**: 保存済みソングブックの一覧を表示

#### ソングブック一覧

```
┌─────────────────────────────────────┐
│ ソングブック                 [ + 保存 ] │
├─────────────────────────────────────┤
│ 🎵 Autumn Leaves               ─ 🗑 │
│    6スケール  最終更新 06/01 14:32   │
├─────────────────────────────────────┤
│ 🎵 Blue Bossa                  ─ 🗑 │
│    4スケール  最終更新 05/28 09:15   │
└─────────────────────────────────────┘
```

※ 最終更新日時は小さいフォントでカード下部に表示。

#### 「ソングブックに保存」フロー

```
「+ 保存」ボタンをタップ
    ↓
確認ダイアログ:
「現在のソングファイルをソングブックに保存します。
 名前を入力してください。」
[ テキストフィールド（空欄・プレースホルダー: "ソングブック名"） ]
    ↓
OK → 現在のソングファイル（localStorage）の内容を D1 に保存
キャンセル → 何もしない
```

※ デフォルト名には日付を入れない（ユーザーが都度入力）。

#### ソングブックの読み込みフロー

```
ソングブックのカードをタップ
    ↓
確認ダイアログ（データ消失警告）:
「"Autumn Leaves" を読み込みます。
 現在編集中のソングファイルは上書きされます。
 よろしいですか？」
    ↓
OK → D1 から読み込み → localStorage に展開 → ソングファイルタブに切り替え
キャンセル → 何もしない
```

⚠️ **読み込みは現在のソングファイルを失う破壊的操作** のため、
確認ダイアログは必須。キャンセルできることを明示する。

---

## 6. 現在編集中ソングファイルの表示（ソングファイルタブ上部）

ソングファイルタブに切り替えたとき、最上部にスケールエディターで
現在編集中のスケールのプレビューをコンパクト表示する。

```
┌─────────────────────────────────────┐
│ 編集中: C メジャーペンタ  [ 登録する ] │  ← 常に最上部に固定
│ [指板プレビュー（小）]               │
├─────────────────────────────────────┤
│ 登録済みスケール一覧...              │
└─────────────────────────────────────┘
```

- 楽器未選択時は表示しない
- 「登録する」ボタンで現在のスケールをソングファイルに追加

## 7. ソングブック数・スケール数の制限

| 制限 | 値 | 理由 |
|------|-----|------|
| ソングブック数/ユーザー | 50件 | UX・DB容量のバランス |
| 1ソングブックあたりのスケール数 | 200件 | UX上の実用上限（D1の技術的制限ではない） |

### D1 の技術的制限について

D1（SQLite）の行サイズ制限は非常に大きく（TEXT列は実質無制限）、
スケール200件のJSONは約100KB程度で全く問題ない。
上記の制限はUX（使いやすさ）のために設けるもの。

---

## 8. 非機能要件

- **ログイン必須**: ソングブックタブはログインユーザーのみ
- **確認ダイアログ**: 保存・読み込みともに必ず確認を取る
- **楽観的UI**: 保存・削除はUI先行で反映し、API失敗時にロールバック
- **エラー表示**: API失敗時はトーストで「保存に失敗しました」等を表示

---

## 9. 受け入れ条件

| # | 条件 |
|---|------|
| AC-01 | タブが「スケールエディター」「ソングファイル」「ソングブック」の3つになる |
| AC-02 | 未ログイン時、ソングブックタブにロック画面が表示される |
| AC-03 | 「+ 保存」でダイアログが出て、名前を入力して保存すると一覧に追加される |
| AC-04 | 保存時に確認ダイアログが出る |
| AC-05 | ソングブックをタップすると確認ダイアログが出る |
| AC-06 | 読み込み後、ソングファイルタブの内容が変わる |
| AC-07 | 削除するとソングブック一覧から消える |
| AC-08 | 各カードに最終更新日時が小さく表示される |
| AC-09 | 別デバイスでログインすると同じソングブックが見える |
| AC-10 | ソングファイルタブ上部に編集中スケールが表示される |
