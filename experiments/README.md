# experiments/ — 技術検証の場（製品コードから隔離）

ここは **技術的な実現可能性を検証する実験場**。製品本体（`src/`）とは**別プロジェクト扱い**で、
本体のルール（ゼロ依存・アーキ制約）や品質ゲートに縛られない、自由に試すための領域。

## 原則

- **製品コードに入れない**: `experiments/` は `src/` から import されない。本体ビルドにも含まれない。
- **本体の品質ゲート対象外**: `npm run build` / `lint`（`eslint src __tests__`）/ `test`（`__tests__/`）/
  デプロイ（`dist/`）はいずれも `experiments/` を見ない。汚さない・汚されない。
- **各実験は自己完結**: 1実験 = 1サブフォルダ。**単体でローカル起動して動く**こと。
- **検証が目的**: 「実機で遅延・精度が出るか」を確かめる。きれいさより**地に足のついた計測**を優先。
- **昇格はあとで**: 検証OKなら、そのとき初めて製品仕様（`docs/`）を書いて `src/` に正式実装する。
  実験コードをそのまま製品に流し込まない（作り直す前提）。

## 実験一覧

| 01 | [note-detection](note-detection/README.md) | マイク入力の**単音ピッチ検出**の遅延・精度を実機検証 | 計画中（コード未着手） |

## 実験のワークフロー

```
アイデア → この下にフォルダ作成 → README に「何を検証するか・成功条件」を書く
       → 単体で動く最小コードを作る（spike）
       → 実機で計測し findings を README に追記
       → 判断: 製品化する/しない/方式変更
          └ する → docs/ に製品仕様を書き src/ に実装（実験はここで役目終了）
```

## 注意

- 隔離されているとはいえ同じ git リポジトリに同居している。完全に切り離したくなったら
  別リポジトリに分離してもよい（その場合この README を移す）。
- 実験ごとに依存を入れたい場合は**その実験フォルダ内で完結**させる（本体の package.json を汚さない）。
  ※ 01 note-detection は**依存ゼロ**（ブラウザ標準APIのみ）で作る方針。
